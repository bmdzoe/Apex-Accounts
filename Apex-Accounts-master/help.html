1. System Overview

Apex Accounts is a web-based accounting system designed for use by administrators, managers, and accountants. The application is intended to be accessible across devices and browsers while maintaining a consistent layout and built-in help throughout the software.

The system supports core accounting and finance functions including chart of accounts, journalizing, ledger review, user management, approval workflows, financial reports, and financial ratio review.

2. User Roles

This section explains what each user role is allowed to do in the system. Use it to understand the difference between administrator, manager, and accountant access.

Administrator

  • Create users and assign roles
  • Update user information
  • Activate or deactivate users
  • Suspend users for a selected date range
  • Approve or reject access requests
  • View all users report
  • View expired passwords report
  • Send email within the system
  • Add, view, edit, or deactivate accounts in the chart of accounts
  • View account event logs

Manager

  • View accounts in the chart of accounts
  • Create journal entries using valid accounts
  • Approve or reject journal entries submitted by accountants
  • Enter rejection comments when rejecting entries
  • View pending, approved, and rejected journal entries
  • Generate, save, email, print, and review financial reports
  • Approve or reject adjusting entries
  • View ledgers and event logs

Accountant

  • View accounts in the chart of accounts
  • Create and submit journal entries
  • Create and submit adjusting journal entries
  • Attach source documents to journal entries
  • View pending, approved, and rejected entries
  • View ledgers and event logs
  • Send email to manager or administrator from account-related pages

4. Login and Security

This section explains the rules for logging in and keeping account information secure.Use it to review password requirements, reset steps, and other security related features.

  • Only registered users can log in.
  • Passwords must be at least 8 characters long.
  • Passwords must start with a letter.
  • Passwords must include a letter, a number, and a special character.
  • Passwords used in the past cannot be reused during password reset.
  • Password information must be stored securely.
  • After three unsuccessful login attempts, the user is suspended.
  • Users receive a notification three days before password expiration.
  • Forgot Password requires email, user ID, and security question verification.
  • All login information is stored in the database.

5. User Management

This section explains how user accounts are created, review, and managed in the system. It also shows what adminstators can do when updating roles, approvals, and account status.

Creating a New User Request

New users can request access by choosing the create account option and entering required personal information such as first name, last name, address, and date of birth. The administrator reviews the request and approves or rejects it.

Username Rules

Usernames are generated using the first name initial, full last name, and a four-digit month-year format based on account creation date.

Administrator User Functions

  • Create users
  • Update existing user information
  • Activate or deactivate users
  • Suspend users for a selected time period
  • Review pending approvals
  • Send email notifications
  • View users reports
  • View expired password reports

6. Chart of Accounts

The Chart of Accounts stores and manages all valid accounts used throughout the system.

Account Information

  • Account name
  • Account number
  • Account description
  • Normal side
  • Account category
  • Account subcategory
  • Initial balance
  • Debit
  • Credit
  • Balance
  • Date/time added
  • User ID
  • Order
  • Statement type
  • Comment

Chart of Accounts Rules

  • Duplicate account names or numbers are not allowed.
  • Account numbers cannot contain decimal points or letters.
  • All monetary values must display two decimal places.
  • Monetary values should display commas when appropriate.
  • Accounts with balances greater than zero cannot be deactivated.
  • Accounts can be made inactive but not deleted.
  • Users can search by account number or account name.
  • Users can filter accounts by tokens such as name, number, category, subcategory, and amount.
  • Clicking an account should open its ledger.

Role Access

  • Administrator: can add, view, edit, and deactivate accounts
  • Manager: can view accounts only
  • Accountant: can view accounts only

7. Journalizing and Ledger

This section explains how journal entries are created, reviewed, and posted to the ledger. It also outlines the rules for balancing entries, apporval status, and ledger activity.

Journal Entry Rules

  • Only accounts found in the chart of accounts can be used.
  • Each transaction must contain at least one debit and one credit.
  • Debits must equal credits before submission.
  • Multiple debits and multiple credits are allowed.
  • Debits appear before credits in each journal entry.
  • Invalid transactions cannot be submitted.
  • Accountants can cancel or reset before submission, but cannot delete after submission.
  • Managers approve or reject submitted entries.
  • Rejected entries require a comment from the manager.
  • Managers receive notification when an entry is submitted for approval.

Journal Entry Status

  • Pending Approval
  • Approved
  • Rejected

Searching and Filtering

  • Search journal entries by account name, amount, or date.
  • Filter entries by date and by status category.

Ledger

  • Approved entries are reflected in the ledger.
  • Each ledger row shows date, description, debit, credit, and balance.
  • The balance after each posted transaction must be accurate.
  • The post reference column is clickable and opens the journal entry that created the ledger record.
  • Users can filter the ledger by date or date range and search by account name or amount.

8. Adjusting Entries and Financial Reports

This section explains how adjusting entries are submitted and how finachial reports are generated from approved data. Use it to understand what users can do with reports and which rules still apply to adjusting entries.

Adjusting Entries

  • Accountants can prepare and submit adjusting journal entries.
  • Managers can approve or reject adjusting entries.
  • Rejected adjusting entries require a comment.
  • Approved adjusting entries update the ledger and financial statements.
  • All normal journal entry rules still apply.

Financial Reports

  • Trial Balance
  • Income Statement
  • Balance Sheet
  • Retained Earnings Statement
  • Cash Flow Statement

Managers can generate, view, save, email, or print financial reports for a selected date or date range.

9. Dashboard and Ratio Analysis

This section explains what information is shown on the dashboard and how ratio analysis is presented to the user. It also helps users understand alerts, summaries, and the meaning of the dashboard color indicators.

  • All logged-in users can view dashboard ratio information where data is available.
  • The dashboard may display important system messages such as entries waiting for approval.
  • Ratios use three color states:
  • Green: good
  • Yellow: warning or borderline range
  • Red: needs closer review

10. Source Documents and Attachments

Source documents can be attached to journal entries and adjusting entries. Supported file types include:

  • PDF
  • Word
  • Excel
  • CSV
  • JPG
  • PNG

11. Event Logs

Event logs record data changes made by users. Each event log stores:

  • Unique event ID
  • User ID of the person making the change
  • Date and time of the change
  • Before image of the record when applicable
  • After image of the record

If an account is newly created, there is no before image. Event logs can be viewed as part of account review.

12. Error Messages

This section explains how error messages are used to help users fix problems in the system. Use it to understand what common errors mean and which issues needs to be corrected.

  • Error messages must be meaningful and help the user fix the issue.
  • Error messages are stored in a database table.
  • Error messages are displayed in red.
  • When the cause of the error is fixed, the error should disappear.

Examples

  • Password does not meet complexity requirements.
  • Debits and credits do not balance.
  • Selected account is not valid.
  • Duplicate account number or account name is not allowed.
  • User ID not found.
  • End date must be after start date.

13. Accounting Reference

This section provides basic accounting information that supports the features used throughout the system. Use it as a reference for equations, balances, statement areas, and other accounting terms.

Accounting Equation

Assets = Liabilities + Equity

Natural Balances

  • Assets normally increase with debits
  • Liabilities normally increase with credits
  • Equity normally increases with credits
  • Revenue normally increases with credits
  • Expenses normally increase with debits

Common Financial Statement Areas

  • Balance Sheet
  • Income Statement
  • Statement of Owners' Equity
  • Cash Flow Statement

Inventory and Costing Topics

  • FIFO
  • LIFO
  • Average Cost
  • Perpetual Inventory
  • Periodic Inventory
  • Cost of Goods Sold

Useful Concepts

  • Matching principle
  • Revenue recognition
  • Consistency
  • Conservatism
  • Materiality
  • Full disclosure
  • Going concern
  • Economic entity
  • Monetary unit
  • Time period assumption
